- **주요 변경**:

  - 디스코드 봇 어댑터(`DiscordAdapter`) 물리적 분리 및 약한 결합 아키텍처 도입.

---

## 🧪 2. 집중 테스트 및 검증 사항 (Testing Agenda)

1.  **통합 명령어 라우팅 검증**:
    - 웹 터미널과 디스코드에서 `/@` (브리핑+검색), `/` (로컬 전용), `/#` (순수 검색)이 의도대로 동작하는지 확인하십시오.
    - 특히 `/` 명령 시 외부 검색 툴(`google_search`)이 차단되고 로컬 위젯 데이터만으로 요약이 되는지 확인하십시오.
2.  **알람(Alarm) 기능 정밀 동기화**:
    - AI가 `[ACTION] SET_ALARM` 태그를 포함한 응답을 보낼 때, `BotManager`가 이를 해석하여 `alarm_service.py`를 통해 실제 알람을 등록하는지 체크하십시오.
    - 홈 서버 대시보드에서 알람 목록이 정상적으로 표시되는지(API 규격 정합성) 확인하십시오.
3.  **글로벌 페르소나 적용**:
    - 사용자 `lang` 설정이 `en`으로 변경되었을 때, AI가 영어 페르소나 지침(`config/i18n/en.json`)을 로드하여 영어로 답변하는지 확인하십시오.

---

## 🛠️ 3. 버그 발생 시 수정 범위 (Fix Scope)

시스템에 "버그쟁이"가 침투했을 경우, 다음 레이어를 우선적으로 점검하십시오.

- **`services/bot_gateway.py` (BotManager)**:
  - 인텐트 해석, 플랫폼별 메시지 변환, 하이브리드 추론 로직의 핵심입니다. 라우팅 오류는 대부분 여기서 발생합니다.
- **`utils.py` > `clean_ai_text()`**:
  - AI 응답의 마크다운 래퍼나 레이블을 제거하는 유틸리티입니다. 파싱 에러(JSON 에러 등) 발생 시 이 함수의 정제 로직을 강화하십시오.
- **`plugins/alarm/router.py`**:
  - 알람 데이터의 API 규격이 위젯과 일치하지 않을 경우 대시보드에서 렌더링 오류가 발생합니다.
- **`config/i18n/*.json`**:
  - 특정 언어에서 AI의 대답이 어색하거나 지침을 어길 경우, 프롬프트 문자열의 누락 여부를 확인하십시오.
