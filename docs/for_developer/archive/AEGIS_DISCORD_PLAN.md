# AEGIS Distributed Interface & Discord Integration Plan (v1.0)

본 문서는 AEGIS 개발팀(Antigravity)이 제안하는 **디바이스 분산형 아키텍처**와 **디스코드 봇 기반 원격 인터페이스**에 대한 상세 설계 계획서입니다. "입력은 디스코드, 출력은 데스크탑 HUD"라는 전제하에 실용성과 보안, 시각적 만족도를 극대화하는 것을 목표로 합니다.

---

## 📅 프로젝트 개요
- **목표:** 데스크탑 HUD의 입력 난제를 디코로 해결하고, 전 세계 어디서든 AEGIS와 소통하는 초연결 환경 구축.
- **주요 원칙:**
    1.  **입력 분리:** 무거운 터미널 입력을 Discord 플랫폼으로 이관.
    2.  **멀티모달 응답:** 디코(텍스트/이미지)와 데스크탑(음성/위젯)의 동시 반응.
    3.  **철저한 개인화:** 지정된 사용자 ID만 접근 가능한 화이트리스트 보안.
    4.  **보안 최우선:** 실시간 화면 캡처(자동 스냅샷) 배제, 고정 에셋 기반 시각화.

---

## 🏛️ 아키텍처 설계 (AEGIS Universal Bot Gateway)

특정 플랫폼(Discord)에 종속되지 않고, 향후 텔레그램, 카카오톡 등 다양한 인터페이스를 손쉽게 추가할 수 있는 **'추상화된 메시징 허브'** 구조를 채택합니다.

### 1. Universal Bot Bridge (Abstract Layer)
- **추상화:** 모든 외부 메시징 플랫폼의 공통 기능을 정의합니다. (`send_text`, `send_image`, `handle_command`)
- **수직 확장:** 플랫폼별 전용 어댑터(DiscordAdapter, TelegramAdapter)만 추가하면 즉시 연동되는 구조입니다.

### 2. AEGIS Messaging Hub (Core Service)
- **역할:** 중앙 관제소. 어떤 플랫폼에서 온 명령이든 AEGIS 내부의 명령 체계(System Call)로 번역하여 전달합니다.
- **라우팅:**
    - `Input Sync`: 외부 봇 ➔ 메시징 허브 ➔ AEGIS 내부 로직 ➔ 데스크탑 HUD 동기화.
    - `Output Sync`: AEGIS 알림 ➔ 메시징 허브 ➔ 활성화된 모든 봇 및 데스크탑 HUD로 멀티캐스팅.

### 3. Desktop HUD (Visual & Audio Persona)
- **역할:** 사용자가 위치한 '현장'의 최종 비주얼 피드백 장치.
- **특징:** 원격(Bot)에서 발생한 모든 이벤트를 음성과 모션으로 즉시 실체화합니다.

---

## 🚥 명령어 라우팅 & 지능형 처리 로직

모든 인터페이스(Discord, Telegram 등)로부터 수신된 입력은 통합된 **3단계 필터링**을 거칩니다.

### 1단계: 플랫폼 전용 커맨드 (Native Command)
- 각 플랫폼의 특화된 기능(디스코드 슬래시, 텔레그램 커맨드 등)을 통해 들어온 명시적 요청.

### 2단계: 유니버설 키워드 인터셉터 (Keywords)
- 플랫폼에 상관없이 공통적으로 인식되는 키워드 매칭.

### 3단계: 통합 AI 의도 분석 (Global AI Reasoning)
- 어떤 채널로 들어오든 동일한 AI 비서(Grok/Gemini)가 컨텍스트를 분석하여 적절한 시스템 액션을 트리거함.

## 🔒 보안 및 리스크 관리

- **Multi-Platform IAM:**
    - `secrets.json`에 각 플랫폼별 허용된 사용자 고유 ID(Discord UID, Telegram ChatID 등)를 관리합니다.
- **One-Voice Sync:**
    - 여러 봇을 쓰더라도 AEGIS의 상태는 하나로 관리되어, 어떤 봇으로 말을 걸든 집의 데스크탑 HUD는 동일한 페르소나로 반응합니다.
- **Critical Action Confirmation:**
    - 데이터 삭제 등 파괴적 동작은 메시징 허브를 통해 전송된 채널에 "정말 실행할까요?"라는 버튼 혹은 메시지를 띄워 최종 컨펌을 받습니다.

## 🚀 단계별 구현 로드맵

1.  **Phase 1: 봇 게이트웨이 기초 설계** (추상 클래스 기반의 `BotManager` 및 기본 인터페이스 정의)
2.  **Phase 2: 첫 번째 어댑터 (Discord)** (계획된 기능을 디스코드 어댑터로 선구현)
3.  **Phase 3: 멀티 플랫폼 확장 테스트** (텔레그램 등 제2의 어댑터를 통한 유연성 검증)
4.  **Phase 4: 시각 보고서 합성 & 통합 푸시** (어떤 플랫폼으로든 예쁜 리포트를 쏠 수 있는 통합 비주얼라이저 연동)

## 💡 기대 효과
- **플랫폼 독립성:** 봇 정책 변화나 신규 플랫폼 등장에 유연하게 대응 가능.
- **UX 일관성:** 어떤 통로를 쓰더라도 AEGIS 대시보드와 데스크탑 HUD는 항상 동일한 사용자 경험을 제공.
- **실용성 극대화:** 내가 선호하는 메신저를 AEGIS의 '조종석'으로 자유롭게 선택하여 사용.
