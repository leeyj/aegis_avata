# AEGIS V2.3 개발 계획서 (Connectivity & Media Sync)

본 문서는 V2.3 로드맵의 각 항목별 구체적인 구현 목표, 예상 난이도 및 개발 우선순위를 정의합니다.

## 📌 V2.3 핵심 과제 통합 테이블

| 항목명 | 구현 핵심 내용 | 난이도 | 예상 우선순위 |
| :--- | :--- | :---: | :---: |
| **의존성 관리자**<br/>(Dependency Manager) | • 플러그인 디렉토리 내 `requirements.txt` 자동 스캔<br/>• 로드 시 누락된 Python 라이브러리 감지<br/>• 터미널 또는 UI를 통한 `pip install` 사용자 승인 및 자동 설치 지원 | 🟡 중 | **1순위** (Highest) |
| **System Tray & Auto-Startup**<br/>(데스크톱 통합) | • `pystray` 또는 `PyQt` 트레이 아이콘 모듈 연동<br/>• 백그라운드 실행 및 시스템 트레이 우클릭 메뉴(종료, 대시보드 열기) 지원<br/>• 윈도우 부팅 시 자동 실행(Auto-start) 레지스트리 또는 시작프로그램 등록 | 🟡 중 | **2순위** (High) |

## 🛠️ 세부 구현 전략 및 리스크 분석

### 1. 의존성 관리자 (우선순위: 1)
*   **배경**: AI 에이전트가 새로운 위젯을 만들 때 자주 외부 라이브러리(`requests`, `bs4`, `tinytuya` 등)를 요구합니다. 이 모듈이 없으면 서버에서 500 에러를 뿜으며 위젯이 뻗는 문제가 발생합니다.
*   **구현 방향**: 시스템 코어(`plugin_registry.py` 등)에서 플러그인을 로드하기 직전, `requirements.txt`가 있다면 `importlib`으로 패키지 유무를 검사합니다.
*   **리스크**: `pip` 설치 시 서버 권한 문제나 OS별 환경 차이 처리 필요.

### 2. System Tray & Auto-Startup (우선순위: 2)
*   **배경**: AEGIS가 단순 웹앱을 넘어 "데스크톱 네이티브 비서(DNA Mode)"로 나아가기 위한 핵심 기능입니다.
*   **구현 방향**: `app_factory.py` 기동 시 별도의 데몬 쓰레드로 시스템 트레이 아이콘을 띄우고, 브라우저가 꺼져도 서버는 데몬 형태로 살아있도록 만듭니다.
*   **리스크**: `flask` 구동 쓰레드와 트레이 아이콘의 GUI 쓰레드 간의 충돌(Main thread 에러)을 주의해야 합니다.

### 3. 로컬 미디어 싱크 (우선순위: 3)
*   **배경**: YouTube 중심의 음악 재생을 넘어, 나만의 음원 라이브러리를 아바타 배경의 비주얼라이저와 완벽하게 동기화하기 위함입니다.
*   **구현 방향**: 프론트엔드에서 음원 스트림을 `AudioContext analyzer` 노드에 연결하여 주파수 데이터를 뽑고, 이 값을 캔버스 애니메이션이나 Live2D 파라미터로 쏴줍니다.
*   **리스크**: 브라우저 보안 정책 상 사용자의 첫 번째 클릭 전에는 AudioContext가 생성되지 않거나 Mute되는 현상 방어(예: 클릭 시 초기화).

### 4. 반응형 그리드 엔진 (우선순위: 4)
*   **배경**: 외출 시 스마트폰으로 접속하여 상태를 보거나 제어하려는 니즈 충족.
*   **구현 방향**: 데스크톱에서는 `position: absolute`로 사용자 커스텀 드래그 배치를 유지하고, 모바일 해상도(width 768px 이하) 감지 시 이를 일괄 해제하고 `grid` 또는 `flex` 컬럼 형태로 위젯들을 차곡차곡 쌓는 스위칭 로직.
*   **리스크**: 드래그 위치 저장 데이터(`settings.json` 내 `positions`)와 반응형 배치가 꼬이지 않도록 명확한 상태 분리(모바일 모드에서는 좌표 저장 무시 등)가 필수.
