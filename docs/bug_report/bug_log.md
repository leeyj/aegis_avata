# AEGIS Bug Report & Resolution Log (v1.8)

이 문서는 모듈화(Plugin-X) 과정에서 발견된 버그들의 발생 상황과 조치 방안을 기록합니다. 유사한 버그의 재발을 방지하는 가이드로 활용합니다.

---

## [2026-03-01]

### 1. 할 일(Todo) 위젯 동기화 UI 및 번역 누락
- **상태**: 새로운 할 일 추가 시 "(Syncing...)" 표시가 사라지지 않고 중복 항목이 보임. 타이틀 한글 번역(`widgets.todo_title`)이 적용되지 않음.
- **원인**: 서버 응답 수신 후 로컬 임시 데이터(`isTemp`)를 제거하는 로직이 미비했으며, i18n JSON 파일에 관련 키가 누락됨.
- **조치**: 
    - `widget.js`의 추가 성공 시점에 `localTasks` 필터링 로직 강화.
    - `i18n.json`에 `widgets` 섹션 추가 및 번역 적용.
    - 하드코딩된 UI 텍스트를 `context._t`로 교체.

### 2. 배경화면 설정 초기화 (저장 동기화 실패)
- **상태**: 배경화면 변경 후 "SAVE SYNC"를 눌러도 새로고침하면 이전 배경(또는 빈 화면)으로 돌아감.
- **원인**: `save_json_config(CONFIG_PATH, data, merge=True)`에서 병합(Merge) 로직이 문제됨. 이전 모드의 필드(예: 비디오 배경의 `is_video: true`)가 남아 이미지 배경 설정을 오염시킴.
- **조치**: 
    - 서버 측 `router.py`에서 `merge=False`로 설정하여 현재 상태만 정밀하게 기록하도록 수정.
    - 클라이언트 측 `index.js`에서 모드 변경 시 관련 플래그 초기화 로직 보강.

### 3. 메인 타이틀 클릭 이벤트 간섭 (브리핑 미작동)
- **상태**: 메인 타이틀 클릭 시 드래그 이동만 발생하고 AI 브리핑(음성 알림)이 실행되지 않음.
- **원인**: 
    - 드래그 임계값(Threshold)이 0으로 설정되어 클릭 시의 미세한 떨림을 모두 드래그로 오인함.
    - Plugin-X 모듈화로 인해 타이틀이 Shadow DOM 내부에 위치하게 되면서 기존 `document.getElementById`가 실패함.
- **조치**: 
    - `ui_drag_manager.js`에 **5px 드래그 임계값** 도입.
    - `ui_briefing_manager.js`에 **Shadow DOM 탐색** 및 **재시도(Retry)** 로직 추가.

### 4. 브리핑 시스템의 논리적 확장성 부족
- **상태**: 메인 타이틀 외의 다른 위젯(뉴스, IoT 등)에서 브리핑 시스템을 사용하기 어려운 구조.
- **원인**: 브리핑 로직이 특정 요소 ID에 하드코딩되어 있고, 플러그인 컨텍스트(`context`)에 API가 노출되지 않음.
- **조치**: 
    - `window.BriefingService` 도입으로 브리핑 로직을 순수 서비스화.
    - `plugin_loader.js`에서 `context.triggerBriefing()` API 공개.
    - 타이틀 위젯을 독립 플러그인화하여 스스로 브리핑을 호출하도록 구조 개선.

## [버그 예방 가이드]
1. **플러그인 접근**: Shadow DOM을 사용하므로 항상 `shadowRoot` 내부를 먼저 탐색하거나, `context` API를 우선 사용하세요.
2. **저장 로직**: 상태가 완전히 전환되는 설정(예: 모드 변경)은 서버 저장 시 `merge=False`를 고려하여 이전 데이터의 잔재를 지우세요.
3. **UX 충돌**: 드래그와 클릭이 동시에 발생하는 영역은 항상 5px 이상의 임계값을 두어 오작동을 방지하세요.
