# AEGIS v1.6~1.8 ➔ v1.9 마이그레이션 가이드 (Clean Install 권장)

기존 1.8 이하 버전에서 1.9 버전(Weather Dynamics & UI Lock 통합)으로의 업데이트는 프로젝트 구조의 안정성과 보안 규격이 강화되었습니다.
과거 파일들과의 충돌(Zombie Files)을 방지하기 위해, 덮어쓰기 업데이트나 `git pull`이 아닌 **기존 데이터를 백업한 뒤 새로 설치(Clean Install)** 하시는 것을 권장합니다. v1.9는 특히 새로운 환경 효과 자산이 추가되었습니다.

---

## 🚀 완벽한 마이그레이션을 위한 3단계 가이드

### 1단계: 기존 중요 데이터 백업하기
현재 사용하시던 `gods` 폴더에서 아래의 항목들을 안전한 곳(바탕화면 등)에 복사하여 보관해 주세요.
(AEGIS는 코어 코드를 제외한 사용자 개인의 모든 인증/로컬 정보가 아래 경로에만 저장됩니다)

* **`config/` 디렉토리 전체**
  * 계정 정보(`secrets.json`), 구글 인증서(`credentials.json`, 토큰 파일들), 날씨/스케줄러 설정 등이 모두 포함되어 있습니다.
* **`models/` 및 'test_models/' 디렉토리 전체** (존재하는 경우)
  * 본인이 개별적으로 추가하여 스튜디오에서 맵핑했던 라이브2D 아바타 모델과 알리아스(Alias) 설정들이 포함되어 있습니다.
* **`headers_auth.txt` 파일** (존재하는 경우)
  * YouTube Music 재생을 위해 저장해두었던 쿠키 정보입니다.

### 2단계: 최신 시스템 새로 내려받기 (Clean Install)
기존에 사용하시던 `gods` 폴더를 과감히 삭제하거나 이름을 다른 것으로 변경(`gods_old`)해 둡니다.
이후 `git clone` 또는 최신 릴리즈 파일을 다운로드하여 새 버전의 `gods` 시스템을 내려받습니다.

### 3단계: 백업 복구 및 실행
1단계에서 따로 복사해 두었던 폴더(`config/`, `test_models/`)와 파일(`headers_auth.txt`)을 새로 내려받은 최신 `gods` 폴더 안에 그대로 **덮어쓰기/붙여넣기** 해줍니다.

이제 터미널에서 `python app.py` (또는 여러분이 구성하신 로컬/홈 서버 실행 명령)를 구동하면 완료됩니다.
마이그레이션 도구나 번거로운 설정 없이 과거의 사용자 설정과 인증 상태가 1.9 버전에 완벽히 연동됩니다! 환영합니다!
v1.9부터는 대시보드 상단 로고 옆의 **자물쇠 아이콘**을 통해 위젯 배치를 고정하거나 해제할 수 있으니 꼭 확인해 보세요.
