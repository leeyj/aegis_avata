# AEGIS Hybrid Routing (@) Specification (v3.7.2)

이 문서는 AI 지능과 플러그인 컨텍스트를 결합한 `/@` (Hybrid) 명령어 체계의 설계 원칙과 구현 가이드를 정의합니다. 인스턴스가 바뀌어도 이 문서를 통해 동일한 맥락에서 개발을 지속할 수 있도록 합니다.

## 1. 하이브리드 라우팅의 정의
- **목적**: 사용자가 특정 플러그인의 데이터(컨텍스트)를 사용하면서도, 정해진 명령어가 아닌 자연어로 질문하거나 복합적인 추론이 필요할 때 사용합니다.
- **포맷**: `/@알리아스 자연어질문` (예: `/@뉴스 오늘 삼성전자 소식 요약해줘`)

## 2. 현재 구현 상태 (As-Is)
- **위치**: `services/bot_gateway/router.py` 내 `handle` 메서드.
- **작동 흐름**:
    1. `@` 접두사 감지 시 `is_hybrid` 플래그 활성화.
    2. 알리아스를 분석하여 `target_plugin_id` 추출.
    3. (우선순위 1) 입력값 중 첫 단어가 해당 플러그인의 **확정적 액션**인지 확인.
    4. (우선순위 2) 확정적 액션이 아니거나 하이브리드 모드인 경우 `run_ai_fallback` 호출.
    5. **프롬프트 구성**: `role_hybrid.md` + 특정 플러그인 데이터 + `restrict_to_plugin_id`.

## 3. 개선 목표 (To-Be)
`/` (Systematic) 체계에서 해결한 이슈들을 `/@`에도 적용해야 합니다.
- **프롬프트 격리**: 다른 플러그인의 액션 가이드가 섞이지 않도록 `restrict_to_plugin_id`를 엄격히 적용.
- **응답 일관성**: 하이브리드 응답에서도 `[DISPLAY]`와 `[VOICE]` 규격을 준수하도록 유도.
- **폴백 로직 정교화**: 알리아스가 없을 경우나 플러그인 데이터가 비어있을 때의 안내 메시지 강화.

## 4. 인수인계 핵심 포인트
- **설명 반복 방지**: 새로운 AI 인스턴스에게 "docs/HYBRID_ROUTING_SPEC.md를 읽고 현재 3단계 명령어 체계에 맞춰 하이브리드 라우팅을 리팩토링해줘"라고 요청하면 됩니다.
- **안전 장치**: 확정적 명령어(`/`)는 건드리지 않고, `is_hybrid` 조건절 내부만 수정하여 리스크를 최소화합니다.
