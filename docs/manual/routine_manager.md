# 🕒 Routine Manager (Briefing Scheduler) 상세 가이드 v1.9

AEGIS 루틴 매니저는 시스템의 모든 자동화 활동과 위젯의 활성 시간을 정밀하게 제어하는 중앙 통제 센터입니다. `v1.9` 업데이트를 통해 직관적인 GUI 에디터와 데이터 기반의 **조건 감시(Condition Watch)** 기능이 통합되었습니다.

---

## 🚀 1. 핵심 기능 (Key Features)

### 🛡️ 게이트키퍼 (Gatekeeper)
특정 위젯이나 기능이 작동할 수 있는 **'허용 시간'**을 정의합니다.
- **주식(Stock)**: 개장 시간에만 알림 허용.
- **메일(Gmail)**: 업무 시간에만 수신 확인.
- **프로액티브(Proactive)**: 수면 시간을 제외한 활동 시간에만 아바타가 먼저 말을 걸도록 제한.
- **편집 기능**: 각 항목의 ✏️ 버튼을 통해 시작/종료 시각(HHMM)과 적용 요일을 수정할 수 있습니다.

### 📅 자동화 루틴 (Routines)
정해진 시간에 특정 액션을 자동으로 실행합니다.
- **정기 브리핑**: 매일 아침 9시에 오늘의 날씨, 금융 지수, 일정을 통합 요약 보고.
- **조건 감시 (v1.8)**: 주가 폭락, 고온 경고 등 특정 데이터 수치가 임계치를 넘으면 즉시 TTS 발화나 액션 실행.
- **미디어 제어**: 특정 시간에 YouTube 음악 재생 또는 정지.
- **배경화면 전환**: 분위기에 맞춰 배경화면(Wallpaper) 자동 변경.
- **범용 액션 (v1.7.1)**: 터미널 명령어(`terminal_command`)나 백엔드 API(`api_call`)를 직접 호출하여 어떤 위젯이든 통제 가능.
- **커스텀 TTS**: 사용자 정의 텍스트를 AI 음성으로 출력. (`{{value}}` 등의 변수 치환 지원)

## 🛠️ 2. 사용 방법 (Workflow)

### 루틴 추가 및 수정
1. 사이드바 메뉴에서 **🕒 루틴 매니저**를 클릭합니다.
2. **+ 추가** 버튼을 누르거나 기존 루틴의 **EDIT** 버튼을 클릭합니다.
3. **Action 드롭다운**을 선택하면 하단에 해당 기능에 대한 상세 설명이 표시됩니다.
4. **Trigger가 '조건 감시(Watch)'일 경우**: 시스템이 플러그인의 `manifest.json > exports` 정보를 자동으로 스캔하여 감시 가능한 센서 목록을 드롭다운으로 보여줍니다. 사용자는 API 주소를 몰라도 센서 이름만 보고 선택할 수 있습니다.
5. 시간과 요일을 설정한 후 **APPLY**를 누릅니다.

### 게이트키퍼 설정 (Active Hours)
1. 상단의 게이트키퍼 항목 우측에 있는 ✏️ 버튼을 클릭합니다.
2. **Logic Type**을 선택합니다:
    - **Allow**: 입력한 시간대에만 작동 허용.
    - **Deny**: 입력한 시간대에는 작동 차단. (예: 새벽에는 루틴이 발동해도 아바타가 말하지 않음)
3. 시간은 `0900`, `1830`과 같은 **HHMM** 형식으로 입력합니다.
4. 적용 요일을 선택하고 **APPLY**를 누릅니다.

### 설정 저장
- 모든 변경 사항은 하단의 **SAVE ALL CONFIGURATIONS** 버튼을 눌러야 서버에 영구적으로 저장됩니다. 저장 시 스케줄러 엔진이 즉시 재시작되어 새 설정이 반영됩니다.

## 🛠️ 3. 기술적 세부사항 (For Power Users)

### 데이터 기반 조건식 (Condition Type System)
루틴 매니저는 플러그인이 선언한 데이터 타입에 따라 지능적으로 비교를 수행합니다:
- **Number**: `>=`, `<=`, `>`, `<` 연산자를 통한 수치 비교 (예: 기온 30도 이상)
- **String**: `==`, `!=` 연산자를 통한 상태 비교 (예: 날씨가 "RAINY" 인지 확인)
- **Boolean**: `true / false` 상태 체크 (예: 시스템 경고등 점등 시)

### TTS 템플릿 변수
`speak` 액션 사용 시 텍스트 안에 다음 변수를 넣으면 실시간 데이터로 치환됩니다:
- `{{value}}`: 현재 측정된 센서 값 (예: 28.5)
- `{{threshold}}`: 사용자가 설정한 기준값 (예: 25.0)
