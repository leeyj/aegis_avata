# 🎨 AEGIS Live2D Studio 가이드 v1.9

AEGIS Live2D Studio는 아바타(Live2D 모델)를 시스템에 적용하기 전, 안전하게 테스트하고 최적화할 수 있는 **전문가용 개발 환경**입니다. `v1.9` 표준에 따라 무제한 커스텀 에일리어스 생성을 지원합니다.

---

## 1. 주요 기능 (Key Features)

### 1.1 모델 브라우저 (Model Browser)
- `test_models/` 디렉토리에 있는 모든 모델을 자동으로 스캔하여 리스트로 보여줍니다.
- 모델을 선택하면 실시간으로 화면에 로딩되어 외형과 기본 동작을 확인할 수 있습니다.

### 1.2 지능형 에일리어스 매니저 (Alias Manager)
- AEGIS 시스템이 아바타의 특정 모션(대기, 기쁨, 슬픔 등)을 명령어로 제어할 수 있도록 파일명을 매핑하는 기능입니다.
- **자동 추론**: `check_assets.py` 엔진을 활용하여 복잡한 일본어/중국어 파일명에서도 적절한 모션을 찾아 자동으로 `alias.json`을 생성합니다.
- **수동 편집**: 자동 매칭이 틀렸거나 세부 조절이 필요할 때 직접 파일을 선택하여 저장할 수 있습니다.
- **💡 커스텀 에일리어스 (Custom Alias)**: 시스템 기본값(`idle`, `joy` 등) 외에 사용자가 원하는 이름(예: `dance`, `wink`)을 자유롭게 정의하여 위젯 플러그인이나 리액션 엔진에서 호출할 수 있습니다.

### 1.3 리액션 시뮬레이터 (Reaction Simulator)
- `reactions.json`에 설정된 다양한 시스템 반응(금융 폭등, 새로운 메일 도착 등)을 테스트합니다.
- 특정 이벤트를 트리거했을 때 아바타가 어떤 표정을 짓고, 어떤 대사를 하며, 어떤 동작을 수행하는지 즉시 확인할 수 있습니다.

### 1.4 원클릭 운영 배포 (Apply to Production)
- 테스트가 완료된 모델을 단 한 번의 클릭으로 `models/` (실제 운영 폴더)로 복사합니다.
- 복사와 동시에 대시보드 메인 아바타로 설정이 변경되어 즉시 적용됩니다.

---

## 2. 작업 워크플로우 (Recommended Workflow)

1.  **모델 준비**: 새로운 Live2D 모델 폴더를 `test_models/` 하위에 넣습니다.
2.  **스튜디오 접속**: 대시보드 사이드바에서 `LIVE2D STUDIO`를 클릭합니다.
3.  **에일리어스 설정**: `Alias Manager` 탭에서 `Intelligent Match`를 실행하거나 수동으로 `idle`, `joy` 등의 모션을 연결하고 `Save Alias`를 누릅니다.
4.  **시뮬레이션**: `Simulator` 탭에서 이벤트를 발생시켜 아바타의 반응이 자연스러운지 확인합니다.
5.  **운영 적용**: 모든 설정이 만족스러우면 `Apply to Dashboard` 버튼을 눌러 실제 비서로 임명합니다.

---

## 3. 관련 파일 구조

- `test_models/`: 개발 및 테스트용 모델 저장소.
- `models/`: 실제 대시보드에서 사용 중인 모델 저장소.
- `alias.json`: 모델 폴더 내부에 생성되며, AEGIS 명령어와 실제 모션 파일(.motion3.json)을 연결하는 지도 역할을 합니다.

---
> [!TIP]
> 모델의 파일명이 너무 난해하여 자동 추론이 안 될 경우, 파일명을 `idle_01.motion3.json`, `joy_01.motion3.json` 식으로 변경하면 엔진이 더 정확하게 찾아냅니다.
